# UI 设计稿 vs 当前代码 差异分析

## 1. Layout 布局 (最高优先级)

### 设计稿特征
- **顶部导航栏**: 毛玻璃效果 `bg-white/60 backdrop-blur-md`，包含品牌名+顶部快捷导航+操作按钮+用户头像
- **侧边栏**: 白色背景，品牌Logo区(蓝色图标+品牌名+副标题)，导航项带图标+大写标签，底部设置/支持链接
- **导航项**: Dashboard / 原始资料 / 成品库 / 元数据 + 设置
- **整体背景**: `bg-slate-50`

### 当前代码
- 无顶部导航栏
- 侧边栏简单: 纯文本Logo "教材资料 CMS"，分组导航(内容管理+Overleaf备份)
- 缺少 Dashboard 路由入口
- 背景 `bg-gray-50`

### 需要修改
- Layout.tsx: 添加顶部导航栏，升级侧边栏视觉
- App.tsx: 添加 Dashboard 路由，默认跳转 Dashboard

## 2. Dashboard 仪表盘

### 设计稿特征
- 渐变背景 `bg-gradient-to-br from-slate-50 to-blue-50/30`
- 4列统计卡片: 原始资料/MinerU解析中/MinIO存储(蓝色渐变)/解析完成
- 卡片样式: `rounded-2xl`, 大字号 `text-4xl`, 图标+标签
- MinIO存储卡片: 蓝色渐变背景+进度条
- 3列主内容: 最近处理(2列宽) + 右侧(处理队列+快速上传+系统状态)
- 快速上传卡片: 蓝色渐变背景+装饰圆形

### 当前代码
- 简单 `p-6` 布局
- 3列统计卡片使用 StatCard 组件
- 2列内容: 最近上传+进行中任务
- 无MinIO存储展示，无快速上传卡片

## 3. SourceMaterialsPage 资料库

### 设计稿特征
- 渐变背景，max-w-[1400px] 居中
- 3列统计卡片(含蓝色渐变存储卡片)
- 搜索栏在白色卡片容器内，更大的输入框
- 网格视图: 3列，卡片带缩略图区域+文件类型标签+状态标签
- 列表视图: 圆角2xl容器，每行带文件图标

### 当前代码
- 简单 `p-6` 布局
- 4列统计卡片(纯白色)
- 搜索栏直接在页面上
- 列表视图: 标准表格
- 网格视图: 简单卡片

## 4. ProductsPage 成品库

### 设计稿特征
- 画廊式布局，3列网格
- 卡片带图片预览区域 `aspect-video`
- Featured 卡片跨2列2行
- 圆形pill筛选按钮
- 下载/分享操作按钮

### 当前代码
- 功能性布局，使用真实store数据
- 卡片无图片预览
- 无Featured概念

## 5. MetadataManagementPage 元数据

### 设计稿特征
- 左右分栏: 左侧资产列表(w-96) + 右侧详情编辑
- 面包屑导航
- 2x2网格元数据卡片(Classification/Governance/Copyright/Tags)
- 资产预览区域
- 开关控件(ToggleLeft/ToggleRight)

### 当前代码
- 标签页布局(tags/rules)
- 表格式管理界面
- 无分栏浏览模式
