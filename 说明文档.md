# EduAsset CMS — 教育资产管理平台 说明文档

## 一、项目规划

### 1.1 项目背景
教育内容资产管理系统（EduAsset CMS），用于管理教育资料从"原始文件"到"可交付成品"的完整生命周期。

核心流水线：**原始资料 → Rawcode（MinerU/OCR解析）→ Cleancode（AI清洗）→ 成品（题库/试卷/讲义等）**

### 1.2 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| React | 18.3.1 | 前端框架 |
| TypeScript | - | 类型安全 |
| Tailwind CSS | 4.x | 样式系统 |
| shadcn/ui (Radix UI) | - | 基础组件库（46个） |
| React Router | 7.x | 路由 |
| Recharts | 2.15.2 | 数据可视化 |
| Lucide React | 0.487.0 | 图标 |
| Vite | 6.x | 构建工具 |
| sonner | - | Toast 通知 |

### 1.3 页面路由

| 路由 | 页面 | 功能说明 |
|------|------|---------|
| `/` | 工作台 Dashboard | 全局概览、统计图表、快速入口 |
| `/source-materials` | 原始资料库 | 资料上传、筛选、管理 |
| `/asset/:id` | 资产详情 | 文件预览、血缘关系、处理历史、版本管理 |
| `/process-workbench` | 处理中心 | 任务流水线监控、审核操作 |
| `/products` | 成品库 | 成品展示、筛选、溯源 |
| `/metadata` | 元数据管理 | 结构化字段、标签、AI规则 |
| `/tasks` | 任务中心 | 全量任务管理、审核台 |

## 二、实施方案

### 2.1 架构设计

采用 **React Context + useReducer** 集中式状态管理：

```
src/
├── store/
│   ├── types.ts          # 所有数据类型定义
│   ├── mockData.ts       # 初始 Mock 数据（来源于原页面硬编码）
│   ├── appReducer.ts     # 状态变更 Reducer（所有 Action 统一处理）
│   └── appContext.tsx    # AppProvider + useAppStore() hook
├── utils/
│   ├── sort.ts           # 排序纯函数（资料、成品排序）
│   └── pagination.ts     # usePagination hook（前端分页）
└── app/
    ├── App.tsx           # 入口：包裹 AppProvider + Toaster
    ├── components/       # 公共组件 + shadcn/ui 组件库
    └── pages/            # 7 个页面组件
```

### 2.2 组件使用规范

| 场景 | 使用组件 | 说明 |
|------|---------|------|
| 弹窗/模态 | `Dialog` (shadcn/ui) | 替换手写 `fixed inset-0` 弹窗 |
| Toggle 开关 | `Switch` (shadcn/ui) | AI 规则启用/禁用 |
| 操作反馈 | `toast()` (sonner) | 所有用户操作均有 toast 提示 |
| 分页 | `usePagination` hook | 统一前端分页逻辑 |
| 筛选 | 本地 `useState` + `useMemo` | 多维度筛选派生计算 |

### 2.3 状态管理约定

- 所有页面数据通过 `useAppStore()` 读取 `state`
- 数据变更通过 `dispatch(action)` 触发
- 支持的 Action 类型见 `src/store/types.ts` 中的 `AppAction`
- 筛选、搜索、排序为页面本地 state，不存入全局 store

### 2.4 函数注释规范

所有函数必须包含：
- 功能描述
- `@param` 参数说明
- `@returns` 返回值说明
