# EduAsset CMS 前端修订总结

> **修订日期**：2026-04-19
> **修订范围**：移除后端 Batch Queue 架构，改为 Material 工作台直驱模式

---

## 一、修订概要

本次修订移除了前端对后端 `batch-queue.mjs` 的依赖，改为工作台列表直驱模式。每个 material 现在独立操作，用户需要在列表中手动点击「解析」和「分析」按钮。

## 三、功能变更说明

### 3.1 批量上传流程

**旧流程**：
1. 用户选择文件
2. 上传到 MinIO
3. 提交到后端队列
4. 自动触发 `/batch/start`（根据 `autoMinerU`、`autoAI` 配置）
5. 后端队列自动轮询处理

**新流程**：
1. 用户选择文件
2. 上传到 MinIO
3. 更新前端 material 状态
4. **停止**，提示用户手动在列表中点击解析

### 3.2 解析/分析操作

**旧方式**：通过后端队列自动处理

**新方式**：
- 在 `SourceMaterialsPage` 表格中添加「处理」列
- 每行显示两个按钮：
  - **解析按钮**：调用 `POST /__proxy/upload/parse/analyze`
  - **AI分析按钮**：调用 `POST /__proxy/upload/ai/analyze`（需先完成解析）
- 按钮状态：
  - 未解析：蓝色「解析」按钮
  - 解析中：蓝色「解析中」按钮（loading）
  - 已解析：绿色「已解析」按钮（禁用）
  - AI 分析类似

### 3.3 状态轮询

**旧方式**：轮询 `/batch/status` 获取队列整体状态

**新方式**：每 3 秒调用 `GET /__proxy/db/materials` 刷新 materials 列表

## 四、UI 变更截图

### 4.1 批量上传弹窗

**变更前**：
- 底部有「自动 MinerU 解析」和「自动 AI 分析」复选框
- 上传完成后自动提交后端队列

**变更后**：
- 移除自动解析复选框
- 上传完成后提示「上传成功，请在列表中点击"开始解析"」

### 4.2 工作台列表

**变更前**：
- 右侧显示 `ServerBatchQueuePanel`
- 阶段列显示「MinerU 解析中」、「AI 分析中」

**变更后**：
- 右侧仅显示系统状态（upload-server、本地 MinerU）
- 新增「处理」列，显示「解析」和「AI分析」按钮
- 按钮根据当前状态动态变化

### 4.3 右下角悬浮按钮

**变更前**：
- 显示后端队列状态：`后端处理中 3/10`
- 优先显示后端队列，后端空闲时才显示前端队列

**变更后**：
- 仅显示前端队列状态：`处理中 2/5`
- 移除后端队列统计逻辑

## 五、后端端点依赖

### 5.1 移除的端点调用

| 端点 | 方法 | 说明 |
|------|------|------|
| `/batch/start` | POST | 自动触发队列处理 |
| `/batch/stop` | POST | 停止队列处理 |
| `/batch/pause` | POST | 暂停队列 |
| `/batch/resume` | POST | 恢复队列 |
| `/batch/status` | GET | 获取队列状态 |
| `/batch/job/:id` | PATCH | 更新任务状态 |
| `/batch/job/:id` | DELETE | 删除任务 |
| `/batch/jobs` | POST | 添加任务到队列 |

### 5.2 新增的端点调用

| 端点 | 方法 | 说明 | 参数 |
|------|------|------|------|
| `/parse/analyze` | POST | 触发 MinerU 解析 | `{ materialId: number }` |
| `/ai/analyze` | POST | 触发 AI 分析 | `{ materialId: number }` |
| `/db/materials` | GET | 获取 materials 列表 | 无 |
